Relaciones y ciclo de vida

Ninguna relación usa RESTRICT — la historia se protege archivando identidades y denormalizando resultados, no candando filas. Por eso la clase de bug "no se puede eliminar" desaparece.

RelaciónCardinalidadAl borrarPor qué
Scenario → Product / Ingredient / Machine / OpResource 1 : N cascade Draft descartable se lleva sus entidades locales
Product → ProductIngredient / Machine / OpRes 1 : N cascade La receta no tiene sentido sin el producto. Sin bloqueo cruzado
Scenario → Optimization 1 : N set null La corrida sobrevive al archivado del escenario (autocontenida)
Product → OptimizationResult 1 : N set null Conserva product_name y canonical_product_id
Optimization → OptimizationResult 1 : N cascade Los resultados pertenecen a la corrida
Identidad con historia (canónica) — archivar Ancla permanente; canonical_id nunca queda colgado
Máquina de estados del escenario. draft (editable) → al optimizar pasa a frozen (inmutable, la foto de los inputs) → editar un frozen forkea un nuevo draft con parent_id. Se archiva, no se borra. Esto reemplaza el input_snapshot JSON del modelo singleton anterior.
